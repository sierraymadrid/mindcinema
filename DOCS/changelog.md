## Cómo actualizar este changelog

- Añadir cambios nuevos primero en `Unreleased`.
- Mover esos cambios a una versión etiquetada cuando se cree el tag correspondiente.
- Mantener solo cambios reales ya terminados, evitando registrar trabajo previsto como si ya estuviera implementado.
